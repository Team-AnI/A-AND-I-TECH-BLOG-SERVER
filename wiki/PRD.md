# PRD

## 1. 제품 목표

- 기술 블로그 게시글 관리 API를 안정적으로 제공
- 에디터/뷰어에서 재사용 가능한 이미지 업로드 API 제공
- 게이트웨이/인증 서버와 연동 가능한 명확한 API 계약 유지

## 2. 핵심 사용자 시나리오

- 작성자는 게시글을 작성하고 임시저장(Draft) 상태로 관리한다
- 작성자는 게시글을 게시(Published) 상태로 전환한다
- 작성자는 게시글을 수정하거나 삭제(soft delete)할 수 있다
- 작성자는 본문에 사용할 이미지를 업로드하고 URL을 받아 사용한다

## 3. 기능 요구사항

- 게시글 생성/상세/목록/수정/삭제 API 제공
- 상태 기반 조회(`Draft`, `Published`) 지원
- 이미지 업로드 타입/용량 검증 제공
- 표준 에러 응답 포맷 및 요청 추적 헤더 제공

## 4. 비기능 요구사항

- 로컬 환경에서 API 단위 및 통합 테스트 가능
- DB 스키마 변경은 Flyway 마이그레이션으로 관리
- 인프라 변경 없이 서비스 API 레이어 중심으로 개발

## 5. 범위 외

- 배포 파이프라인 및 인프라 구조 변경
- 인증/인가 서버 내부 정책 구현
